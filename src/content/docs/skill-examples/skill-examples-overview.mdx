My [writing skill](/docs/writing-samples/writing-samples/) is a subset of what I do as a technical writer/documentation engineer. Here are some example projects and demos that show some of my technical skills and knowledge.

## MCP Markdown server

* [Github repo](https://github.com/pwitcher/mcp-markdown-server)
* [Readme](https://github.com/pwitcher/mcp-markdown-server/blob/main/README.md)

In this example project I created an MCP server that enables Claude Desktop to parse and query Markdown files in a local directory. It demonstrates my understanding of MCP servers and how MCP provides an interface for LLM applications to parse, query, and audit Markdown documents. Hypothetically, this could be extended to audit Markdown files for SEO keywords, style guide violations, and sensitive data like API keys.

## This website

* [Github repo](https://github.com/pwitcher/far-force)
* [Readme](https://github.com/pwitcher/far-force/blob/main/README.md)

I built www.petewitcher-techwriter.com using [Astro](https://astro.build) and the [Starlight](https://starlight.astro.build) documentation framework, writing Markdown and MDX in VSC. I decided to host the site on GitHub Pages with a GitHub Actions CI/CD pipeline. I automated the pipeline so every push to the main branch builds and publishes the latest version. This project demonstrates my documentation engineer skills, like managing a docs-as-code workflow and configuring an automated build-and-deploy pipeline. I worked with Claude Code to configure the Astro site and the GitHub Action.
