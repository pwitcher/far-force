My [writing skill](/docs/writing-samples/writing-samples/) is a subset of what I do as a technical writer/documentation engineer. Here are some example projects and demos that show some of my technical skills and knowledge.

## MCP Markdown server

* [Github repo](https://github.com/pwitcher/mcp-markdown-server)
* [Readme](https://github.com/pwitcher/mcp-markdown-server/blob/main/README.md)

In this example project I created an MCP server that enables Claude Desktop to parse and query Markdown files in a local directory. It demonstrates my understanding of MCP servers and how MCP provides an interface for LLM applications to parse, query, and audit Markdown documents. Hypothetically, this could be extended to audit Markdown files for sensitive data like API keys or style guide violations.

## This website

* [Github repo](https://github.com/pwitcher/far-force)
* [Readme](https://github.com/pwitcher/far-force/blob/main/README.md)

I built www.petewitcher-techwriter.com using [Astro](https://astro.build) and the [Starlight](https://starlight.astro.build) documentation framework, with all content authored in Markdown and MDX. The site is hosted on GitHub Pages and deployed automatically through a GitHub Actions CI/CD pipeline, so every push to the main branch builds and publishes the latest version. This project demonstrates my ability to work as a documentation engineer: structuring a content-driven site, managing a docs-as-code workflow with version control, and configuring an automated build-and-deploy pipeline — the same tooling and practices I'd bring to a production documentation set.
