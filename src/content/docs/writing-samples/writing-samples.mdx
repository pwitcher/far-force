#### Create an MCP server with Postman Flows
[PDF](https://drive.google.com/file/d/1HVCGgEpouWCDUpmmavkZDagYcRmM4Gsf/view?usp=drive_link) | [Live Postman user docs](https://learning.postman.com/docs/postman-ai/mcp-server-flows/create-mcp-server-flow)

A step-by-step **procedure** that demonstrates how to build and deploy an LLM-agnostic Model Context Protocol (MCP) server using low-code visual workflows, data parsing, and JSON schemas.

#### Flows CLI commands
[PDF](https://drive.google.com/file/d/15B-2UfWKHvVLMTQG-NKhlexA9FvJ59ZJ/view?usp=drive_link) | [Live Postman user docs](https://learning.postman.com/docs/postman-cli/postman-cli-flows)

A structured CLI **reference** guide for running, deploying, and debugging automated workflows programmatically and from the terminal. To validate these commands and options I downloaded and installed the canary build of the Postman CLI. I created a test environment with collections, requests, and other assets. Then I tested each command and option and verified the result. Finally, I compared the canary build's man pages with the documented commands and options.
